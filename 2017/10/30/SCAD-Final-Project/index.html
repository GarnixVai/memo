SCAD Final Project

Posted on 2017-10-30   |   In assignment   |  

組名：True Fetech

組員：102062340蔡和容、102062108徐芷翎、102062314朱凰華、102062120簡欣妤

:::info
Blossoming Tree是一個以社會議題為基本單位的新聞主題視覺化網站，使用者可以在這個平台上建立自己關注的新聞議題樹，並與朋友分享，統籌贊成或反對等意見。甚至還推薦使用者沒有關注到的社會議題、及使用者關注的社會議題中忽略的新聞。
:::

設計及目的

Blossoming Tree幫助關注社會議題的使用者可以有結構性的組織並追蹤他所關注的社會議題，並可以與朋友進行討論。在一個新聞議題討論間中，一個新聞節點的顏色深淺由這個新聞討論間成員的點擊閱覽次數決定，而一個新聞節點的大小則由這個新聞本身的轉推數決定，這樣的視覺化呈現方式可以讓使用者更好的選擇自己想看的新聞。Blossoming Tree也希望通過大量使用者的使用，收集不同群體對於特定社會議題的偏好與支持度，以期可以推薦使用者非同溫層的新聞議題。

流程圖

特色功能

  1. 根據一則新聞網址建一個新聞討論間
    • 使用FB登入建立使用者資料
    • 每個新聞討論間都有獨立網址，以方便使用者邀請朋友一起討論
  2. 新聞討論間的功能一：資料視覺化
    • 一則新聞即為一個節點
    • 由房間成員點擊次數決定節點顏色深淺
    • 由新聞本身的轉推數決定節點大小
    • 節點上會顯示該則新聞的贊成/不贊成比例
  3. 新聞討論間的功能二：自動推薦
    • 使用者點擊澆水圖示後，系統會推薦近期相關新聞：
    • 系統會根據其他新聞討論間的資料，推薦使用者沒有關注到的社會議題
    • 系統會根據其他新聞討論間的資料，推薦使用者關注的社會議題中忽略的新聞
  4. 新聞討論間的功能三：社群互動
    • 使用者可以在特定新聞下留言或下關鍵字
    • 使用者可以表示對一則新聞贊成或不贊成

方法與技術

  1. 新聞議題樹的資料視覺化?
    我們使用D3.js的Force Layout來動態配置新問議題樹所有節點的顯示位置。Force Layout將視覺元素視為網路中一個個的節點，彼此有斥力、全體也有一個引力，隨著程式的運行，節點在各種力的交互作用下會逐漸收斂到一個穩定位置。

  2. 如何找到近期熱門新聞?
    我們使用Twitter API來抓取特定幾個新聞平台最近200則貼文，並且從每個帳號的200則近期貼文選出50個轉推數最高的新聞，然後每個帳號的50個新聞貼文再綜合起來照轉推數排序後回傳。

  3. 如何根據現有節點推薦相關(相似)新聞?
    我們把從推特拿取的新聞作為候選，再加上網頁已知的現有節點)的URL，經過各種新聞內文處理(Beautiful soup清除標籤及Script並過濾註解和內文過多相似的字詞等)後傳給專門計算兩則文章相似度的函式，將基準點與候選新聞的相似度(此利用到sklearn的TF-IDF)存於一陣列後回傳，以0.4為門檻，符合標準的結果就將會做為推薦新聞。 值得注意的是除了beautifulsoup library ，我們還用到newspaper3去擷取每則新聞的最佳圖片以及關鍵字，作為之後所顯示的標籤以及縮圖。

  4. 如何根據一篇新聞網址建立一個新聞討論間?
    我們根據使用者輸入的新聞網址，查找其新聞標題及內容後，從近日相關新聞中推薦兩到三則新聞，再將這幾則新聞存成一個以時間及亂數為名的JSON檔案，然後在資料庫中新增一個同名的資料表，最後跳轉到一個從後端拿去該字料表及該JSON檔案的網址。

  5. 如何根據其他新聞討論間的資料，推薦使用者沒有關注到的社會議題?
    系統會從眾多新聞議題樹所整理而成的資料庫中，查找特定新聞討論間中所沒有的關鍵字，並將這些關鍵字及其新聞標題與連結一併顯示推薦給使用者。

  6. 如何根據其他新聞討論間的資料，推薦使用者關注的社會議題中忽略的新聞?
    系統會從眾多新聞議題樹所整理而成的資料庫中，查找特定新聞討論間中所存在的關鍵字(即社會議題)，並將這些找到的新聞進行篩選，只顯示特定新聞討論間中所不存在的新聞，推薦給使用者。

未來發展

  1. 在解決同溫層問題時，系統根據其他新聞討論間的資料，推薦使用者關注的社會議題中忽略的新聞論點。但這個功能需要多人使用blossoming tree效果才會比較明顯，也能比較準確地根據實作結果進行改善。

  2. 計算新聞相似度及尋找新聞中的關鍵字等方法，我們目前是針對英文新聞來進行應用。如何將方法延伸至中文新聞的應用，是未來要面對的課題。
